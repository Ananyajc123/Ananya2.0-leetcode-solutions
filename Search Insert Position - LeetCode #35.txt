Concept: Binary Search Variation
Key Idea:

Standard binary search to find target
If not found, left pointer gives insert position
After loop, left is where target should be inserted

Pattern: Binary Search
Time: O(log n)
Space: O(1)

javaclass Solution {
    public int searchInsert(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;
            
            if (nums[mid] == target) {
                return mid;  // Found target
            } else if (nums[mid] < target) {
                left = mid + 1;  // Search right half
            } else {
                right = mid - 1;  // Search left half
            }
        }
        
        // Target not found, left is the insert position
        return left;
    }
}

ğŸ’¡ KEY POINTS

Why does left give insert position?

When target not found, left ends up at first element > target
This is exactly where we should insert to maintain sorted order


Magic of binary search:

If target exists â†’ returns its index
If target doesn't exist â†’ returns where it should be
